# 网页版小说跟打器开发文档

## 项目概述

本项目是一个网页版的小说跟打练习工具，采用创新的输入方式：将用户输入区域与小说文本显示区域合并，在用户输入光标后方以待输入文本的形式显示剩余小说内容。

**当前版本**：v0.2.0

## 核心特性

### 1. 创新输入体验
- **光标后跟随显示**：用户输入光标后方显示待输入的小说文本
- **遮罩/高亮机制**：已输入部分正常显示，未输入部分以浅色/灰色显示
- **实时比对**：输入时实时比对用户输入与原文，错误时给予视觉反馈
- **IME 组合内容隐藏**：中文输入法选字时不显示拼音，避免干扰视线

### 2. 文本显示逻辑
- 显示窗口：始终在光标周围显示固定长度的文本（如前后各 50 字）
- 自动滚动：随着输入进度自动滚动显示内容
- 段落支持：支持多段落小说文本

### 3. 打字统计功能
- 打字速度（WPM）
- 准确率
- 用时统计
- 错误数统计
- 正确字数统计
- 进度条实时显示

### 4. 章节管理功能 ✅ 已完成
- 支持 `.txt` 格式文件上传
- 智能识别章节结构（"第 X 章"格式）
- 章节列表展示与选择
- 章节切换功能
- 支持快捷键（完成状态下按任意键进入下一章）

### 5. 进度保存系统 ✅ 已完成
- **每本小说只保留一个进度**（包含当前章节索引和光标位置）
- 完成一章后自动保存到下一章开头
- 刷新页面后自动恢复进度
- 实时保存打字统计数据（用时、错误数、正确字数）
- LocalStorage 持久化存储

### 6. 小说管理功能 ✅ 已完成
- 上传的小说自动保存到本地
- 已保存小说列表展示
- 点击即可加载已保存的小说
- 支持删除已保存的小说

## 技术选型

### 最终技术栈
- **构建工具**：Vite
- **框架**：Vue 3（Composition API + `<script setup>` 语法）
- **语言**：TypeScript
- **样式**：原生 CSS（CSS 变量实现主题）
- **实现思路**：方案 A（contenteditable div）
- **数据存储**：LocalStorage

### 方案 A 详细说明

使用 `contenteditable` 的 div 作为输入容器：

```
┌────────────────────────────────────────────────┐
│ contenteditable div                            │
│ ┌──────────┬───────┬──────────────────────┐   │
│ │ 已输入文本 │ 光标  │ 待输入文本 (灰色)    │   │
│ │ (normal)  │ (_)   │ (pending style)     │   │
│ └──────────┴───────┴──────────────────────┘   │
└────────────────────────────────────────────────┘
```

**优点**：
- 原生光标行为，无需手动实现
- 天然支持文本选择、复制粘贴
- 移动端兼容性较好

**挑战与解决方案**：
| 挑战 | 解决方案 |
|------|----------|
| 需要控制可输入内容 | 监听 input 事件，限制输入长度不超过原文 |
| 光标位置同步 | 使用 Selection + Range API 定位光标 |
| IME 中文输入 | 监听 compositionstart/compositionend 事件 |
| 样式分离渲染 | 使用 `<span>` 标签分段包装不同状态的文本 |

#### 文本分段渲染
将文本分为三段进行不同样式的渲染：
1. **已输入区域** - 正常样式
2. **光标位置** - 闪烁光标指示器
3. **待输入区域** - 灰色/浅色样式

## 关键技术点

### 1. Vue 3 组件设计

**TypingEditor.vue 核心逻辑**：
```vue
<script setup lang="ts">
import { ref, computed, watch } from 'vue'
import { useStats } from '@/composables/useStats'

const props = defineProps({
  originalText: String
})

// 使用 contenteditable div
const editorRef = ref(null)
const isComposing = ref(false) // IME 输入状态
const cursorPosition = ref(0)

// 文本分段渲染
const typedText = computed(() => props.originalText.slice(0, cursorPosition.value))
const pendingText = computed(() => props.originalText.slice(cursorPosition.value))

// 监听输入事件
const handleInput = (e) => {
  if (isComposing.value) return // IME 选字中不处理
  // 比对输入内容
  // 更新光标位置
  // 触发统计更新
}
</script>
```

### 2. 光标位置控制
- 使用 `window.getSelection()` 和 `Range` API
- 使用 `document.createRange()` 精确定位
- `setCursorPosition()` 方法支持进度恢复

### 3. 中文输入处理
- 处理 IME 输入状态（compositionstart/compositionend）
- 避免在中文输入过程中进行比对
- **IME 组合内容隐藏方案**（v0.1.0）：
  - 使用 `color: transparent` 隐藏 cursor span 中的 IME 输入内容
  - 使用 `::before` 伪元素显示光标字符 `|`
  - 设置 `width: 1ch` 固定光标宽度
  - 设置 `white-space: nowrap` 防止 IME 内容换行影响布局

### 4. 进度保存策略

```typescript
interface TypingProgress {
  chapterIndex: number        // 当前章节索引
  cursorPosition: number      // 在当前章节中的光标位置
  elapsedTime: number         // 累计用时
  errors: number              // 错误数
  correctChars: number        // 正确字符数
  completedAt?: string        // 完成时间（如果已完成）
}

// 每本小说只保留一个进度
// key 是小说标题，value 是进度数据
interface NovelProgress {
  [key: string]: TypingProgress
}
```

## 注意事项

1. **IME 输入处理**：中文输入法需要特殊处理，避免在选字阶段进行比对
2. **移动设备兼容**：当前主要面向桌面端，移动端可作为后续扩展
3. **性能优化**：长文本渲染时注意虚拟滚动，避免 DOM 节点过多
4. **可访问性**：考虑键盘导航和屏幕阅读器支持
5. **LocalStorage 限制**：单域名下约 5MB 存储空间，适合存储文本和进度数据
